# bottom-up 分析模式 — 自底向上漏洞分析

## 触发条件

当任务书中 `分析模式` 为 `自底向上` 时使用。

## 派发的 Subagents

此模式按以下顺序派发 subagent 协作完成分析：

1. **explore** — 接收入口目标，扫描项目找到所有目标操作（如所有日志打印、所有 exec 调用），产出目标清单
2. **vulnerability-analyst** — 基于目标清单，逐条分析数据源和利用条件，产出漏洞判定

## 工作步骤

### 1. 定位目标

- 找到用户指定的目标位置
- 确认目标的类别：SQL 查询 / 命令执行 / 文件操作 / 网络请求 / 反序列化 / 模板渲染 / 配置项 / 依赖版本 / 凭据字符串 / 其他
- 记录目标的关键参数或值

### 2. 分析深度（按需，不强制）

从目标出发向调用上游看，但不强制追踪到外部输入源。

- 如果目标自身就能判断安全与否（配置值、版本号、硬编码密钥）→ 无需向上追踪，直接跳到步骤 3
- 如果目标的部分参数来自上层变量 → 向上追溯几层，直到能做出合理判断为止
- 如果目标参数明显来自外部输入 → 继续向上完整追踪到 source
- 如果追踪两三层后上下文仍不足以判定 → 基于已有证据给出风险评级，标注信息缺口

不要为了完整而冗余回溯。够用就停。

### 3. 安全分析

- 如果目标涉及代码调用（SQL/命令/文件/网络），检查：
  - 参数是否可控、是否有安全防护（过滤、转义、参数化查询等）
  - 防护是否可以被绕过
- 如果目标涉及配置/依赖/凭据，检查：
  - 配置值是否违反安全基线（如 debug=true、权限过大）
  - 依赖版本是否包含已知 CVE
  - 凭据是否为硬编码或权限过大

### 4. 结论判定

- 证据充分 → **TRUE POSITIVE** / **FALSE POSITIVE**
- 信息不足以判定 → **需人工确认**，标注缺失信息
- 风险等级：严重 / 高危 / 中危 / 低危

### 5. 输出格式

```
## 分析结果: <目标位置>

### 目标信息
- 位置: <文件路径:行号>
- 名称: <函数名/配置项/依赖名/...>
- 类别: <命令执行/SQL注入/文件写入/配置风险/...>

<如有数据流回溯，加入此节>
### 数据流回溯
<目标参数> ← <函数A> ← <函数B> ← <输入源X>
路径说明: <每一步的数据变换说明>

### 判定
- 结论: <TRUE POSITIVE / FALSE POSITIVE / LOW RISK / 需人工确认>
- 风险等级: <严重/高危/中危/低危>
- 判断依据: <关键证据>
```
