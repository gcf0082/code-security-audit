# Agent 索引

## top-down

- **适用场景**: 用户指定了入口点——API 接口、handler、CLI 命令、导出函数、模块/目录的攻击面。
- **场景判断原则**: 任务涉及"xxx 接口/handler/入口/模块是否安全"或"梳理 xxx 的攻击面"→ top-down。
- **适用举例**:
  - "审计 /api/v2/user/register 接口的 SQL 注入问题" → 接口入口
  - "分析 admin 模块中所有 handler 的权限校验是否完善" → handler 入口
  - "审一下 src/services/export.py 有没有安全问题" → 文件级入口枚举
  - "对 /app/api 目录做攻击面分析，找出所有外部入口" → 目录级入口枚举
  - "看看这个 CLI 命令会不会有参数注入" → CLI 入口
- **不适用举例**:
  - "检查日志打印有没有泄漏 token" → 非入口分析，top-down 不处理
  - "确认代码中 exec() 调用能否被外部利用" → 非入口分析，top-down 不处理
- **能力简介**: 入口函数识别、调用链追踪、参数传播分析、攻击面映射、漏洞定位。

## bottom-up

- **适用场景**: 用户关注某类操作是否安全——日志打印、网络请求、文件操作、命令执行、SQL 查询等。此类操作本质是 sink 点。
- **场景判断原则**: 任务涉及"xxx 操作是否安全/有没有泄露/会不会被利用"→ bottom-up。主 skill 只需识别出操作类型，不用先找到具体代码行。
- **适用举例**:
  - "分析 /app/services 目录下日志打印有没有敏感信息泄露" → 日志函数是 sink
  - "检查项目中所有文件写入操作是否存在路径穿越风险" → 文件写入是 sink
  - "看这个项目有没有往外发 HTTP 请求，会不会泄露数据" → 网络请求是 sink
  - "确认 lib/db.py:127 的 cursor.execute() 能否 SQL 注入" → 指定了具体 sink 行
  - "跑一下 Semgrep 扫出来的告警帮我确认是不是误报" → 告警定位到 sink
- **不适用举例**:
  - "审计 /api/v2/user/register 接口的 SQL 注入问题" → 非操作安全检查，bottom-up 不处理
  - "查看 admin 模块的认证逻辑是否完善" → 非操作安全检查，bottom-up 不处理
- **能力简介**: Sink 点分类、反向数据流追踪、输入源识别、利用条件判断、误报排除。
