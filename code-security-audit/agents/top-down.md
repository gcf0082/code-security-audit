# top-down Agent — 自顶向下漏洞分析

## 触发条件

当任务书中 `分析模式` 为 `自顶向下` 时使用。

## 工作步骤

### 1. 理解入口

- 确定入口函数/接口的准确位置和签名
- 识别入口的参数来源（HTTP 参数、CLI 参数、环境变量、消息队列等）
- 如果入口是一个目录，先用 glob/search 发现该目录下所有外部可调用入口

### 2. 调用链追踪

- 从入口出发，按调用关系逐层向下追踪
- 记录每一层的关键参数传递和变换
- 识别分支逻辑（权限校验、输入校验、格式转换等）
- 标注所有到达不可信外部函数（sink）的路径

### 3. 安全检查点评估

沿途检查以下安全控制是否存在、是否有效：

- 身份认证与鉴权
- 输入验证与过滤
- 参数化查询 / ORM 使用
- 输出编码
- CSRF/XSS 防护
- 文件路径校验
- 命令注入防护
- 反序列化安全

### 4. 漏洞定位

- 对每条到达 sink 的路径，判断外部输入是否可控且未充分防护
- 给出具体漏洞类型（sqli/rce/ssrf/xxe/path-traversal/idor 等）
- 提供调用链路径作为证据

### 5. 输出格式

```
## 分析结果: <接口/函数名称>

### 入口信息
- 位置: <文件路径:行号>
- 参数来源: <HTTP/CLI/其他>

### 调用链
<入口> → <函数A> → <函数B> → <sink>

### 发现的问题
- [高危] <漏洞描述>
  - 类型: <漏洞分类>
  - 位置: <文件路径:行号>
  - 可控路径: <从入口到 sink 的数据流>
  - 利用条件: <需要满足的前提>
```
