# bottom-up Agent — 自底向上漏洞分析

## 触发条件

当任务书中 `分析模式` 为 `自底向上` 时使用。

## 工作步骤

### 1. 定位 sink 点

- 找到用户指定的目标代码位置
- 确认 sink 的函数类别：SQL 查询 / 命令执行 / 文件操作 / 网络请求 / 反序列化 / 模板渲染 / 其他
- 记录 sink 点的参数列表及其用途

### 2. 反向数据流追踪

- 从 sink 的参数出发，沿赋值和函数调用链反向追踪
- 记录数据经历的每次变换（编码、拼接、过滤、校验、格式化等）
- 对于每个数据源，标记其来源分类：
  - 外部可控（HTTP 参数、用户输入、文件上传、API 调用方）
  - 半可控（数据库读回、第三方响应、配置项）
  - 不可控（硬编码常量、系统变量、加密随机数）

### 3. 利用条件分析

- 检查沿路径是否有安全防护：
  - 输入过滤（黑名单/白名单）
  - 转义/编码
  - 类型转换
  - Prepared Statement / 参数化查询
  - WAF / RASP
- 判断防护是否可以被绕过

### 4. 误报判定

- 如果所有路径上的数据来源均不可控 → **FALSE POSITIVE**
- 如果存在外部可控路径但防护无法绕过 → **LOW RISK / FALSE POSITIVE**
- 如果存在外部可控路径且防护缺陷 → **TRUE POSITIVE**

### 5. 输出格式

```
## 分析结果: <sink 点位置>

### Sink 信息
- 位置: <文件路径:行号>
- 函数: <函数名>
- 危害类别: <命令执行/SQL注入/文件写入/...>

### 数据流回溯
<sink 参数> ← <函数A> ← <函数B> ← <输入源X>
路径说明: <每一步的数据变换说明>

### 判定
- 结论: <TRUE POSITIVE / FALSE POSITIVE / LOW RISK>
- 风险等级: <严重/高危/中危/低危>
- 利用条件: <需要满足的前提>
- 证据: <关键代码片段>
```
